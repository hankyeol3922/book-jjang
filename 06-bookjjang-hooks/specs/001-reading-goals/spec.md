# Feature Specification: 월별 독서 목표 (Monthly Reading Goals)

**Feature Branch**: `001-reading-goals`

**Created**: 2026-07-20

**Status**: Draft

**Input**: User description: "사용자가 월별 독서 목표(권수)를 설정하고, 홈에서 이번 달 달성률을 확인하는 기능" (assistant가 사용자 요청 흐름에 따라 기본 기능으로 선정)

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 월별 목표 설정 (Priority: P1)

사용자가 이번 달에 읽을 책 권수를 목표로 정한다. 목표를 정하는 것만으로도 "이번 달엔 N권 읽겠다"는 약속이 생겨 독서 동기를 부여받는다.

**Why this priority**: 목표가 존재해야 이후의 달성률 표시·피드백이 의미를 가진다. 이 기능 하나만 있어도 "목표를 세운다"는 독립적 가치를 준다.

**Independent Test**: 로그인(또는 익명) 상태에서 목표 설정 화면에 진입해 권수를 입력·저장하고, 다시 들어왔을 때 저장된 목표가 그대로 보이는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 이번 달 목표가 없는 사용자, **When** 목표 권수로 10을 입력하고 저장하면, **Then** 이번 달 목표가 10권으로 저장되고 확인된다.
2. **Given** 이번 달 목표가 10권인 사용자, **When** 목표를 15권으로 수정하면, **Then** 목표가 15권으로 갱신된다.
3. **Given** 목표 입력 화면, **When** 0·음수·소수 등 유효하지 않은 값을 입력하면, **Then** 저장이 거부되고 안내 메시지가 표시된다.

---

### User Story 2 - 홈 달성률 표시 (Priority: P2)

홈 화면에서 이번 달 목표 대비 지금까지 완독한 권수와 달성률(%)을 한눈에 확인한다.

**Why this priority**: 목표(US1)를 세운 사용자에게 진행 상황을 되돌려주어 지속 동기를 만든다. US1이 있어야 표시할 대상이 생긴다.

**Independent Test**: 목표가 설정된 사용자가 책 몇 권을 "다 읽음"으로 표시한 뒤 홈에 진입해, `완독 권수 / 목표 권수`와 백분율이 올바르게 표시되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 목표 10권·완독 4권인 사용자, **When** 홈에 진입하면, **Then** "4 / 10권 · 40%"가 표시된다.
2. **Given** 목표가 설정되지 않은 사용자, **When** 홈에 진입하면, **Then** 달성률 대신 "이번 달 목표를 설정하세요" 안내가 표시된다.
3. **Given** 목표 10권·완독 12권인 사용자, **When** 홈에 진입하면, **Then** "12 / 10권 · 목표 초과 달성"이 표시된다.

---

### User Story 3 - 목표 결과 피드백 (Priority: P3)

한 달이 끝나거나 목표를 달성한 순간, 사용자에게 축하/회고 메시지를 보여준다.

**Why this priority**: 성취감과 다음 달 목표 설정을 유도하는 부가 가치. 핵심 흐름(US1·US2) 없이는 성립하지 않는다.

**Independent Test**: 완독 권수가 목표에 도달한 시점, 또는 새 달로 넘어간 직후 홈에서 해당 피드백 문구가 나타나는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 목표 10권을 달성한 사용자, **When** 홈에 진입하면, **Then** 달성 축하 메시지가 표시된다.
2. **Given** 지난달 목표 미달(7/10)로 새 달이 시작된 사용자, **When** 홈에 진입하면, **Then** 지난달 회고와 이번 달 목표 설정 유도가 표시된다.

---

### Edge Cases

- 목표를 0, 음수, 소수, 공백으로 입력하면? → 거부 + 안내(FR-002).
- 완독 권수가 목표를 초과하면 달성률이 100%를 넘는데 어떻게 표시? → 실제 권수 + "목표 초과 달성"(FR-007).
- 월 중간에 목표를 낮췄더니 이미 달성 상태가 되면? → 현재 목표 기준으로 즉시 재계산.
- 목표를 세우기 전 이미 완독한 책들은 이번 달 집계에 포함되나? → 완독 "일자"가 해당 월이면 포함(목표 설정 시점 무관).
- 비회원(익명) 사용자가 목표를 세운 뒤 정식 로그인하면 목표가 이어지나? → v1 범위: 익명 세션에 저장, 계정 병합은 별도 기능(Assumptions).
- 월이 바뀌면 지난달 목표/진행은? → 지난달 기록은 보존, 새 달은 미설정으로 시작(FR-009).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사용자는 특정 연월(YYYY-MM)에 대해 목표 권수를 설정할 수 있어야 한다.
- **FR-002**: 목표 권수는 1 이상의 정수여야 하며, 시스템은 0·음수·소수·비어있는 값을 거부하고 안내를 제공해야 한다.
- **FR-003**: 사용자는 현재 월의 목표를 언제든 수정할 수 있어야 한다.
- **FR-004**: 시스템은 사용자가 "다 읽음"으로 표시한 책 중 해당 월에 완독된 권수를 자동 집계해야 한다.
- **FR-005**: 홈 화면은 현재 월의 완독 권수, 목표 권수, 달성률(%)을 표시해야 한다.
- **FR-006**: 현재 월 목표가 없을 때 홈은 달성률 대신 목표 설정을 유도하는 안내를 표시해야 한다.
- **FR-007**: 완독 권수가 목표를 초과해도 실제 완독 권수와 함께 "목표 초과 달성" 상태를 정확히 표시해야 한다.
- **FR-008**: 목표와 진행 상황은 사용자(계정 또는 익명 세션)에 영구 저장되어 재접속 시 유지되어야 한다.
- **FR-009**: 월이 바뀌면 새 월의 목표는 미설정 상태로 시작하며, 지난달 목표·완독 기록은 보존되어야 한다.
- **FR-010**: 목표 달성 시점 또는 새 달 진입 시, 시스템은 그에 맞는 피드백 메시지를 표시해야 한다.

### Key Entities *(include if feature involves data)*

- **독서 목표(ReadingGoal)**: 특정 사용자의 특정 연월에 대한 목표. 속성 — 대상 사용자, 대상 연월(YYYY-MM), 목표 권수, 생성/수정 시각. 사용자×연월 조합당 하나.
- **완독 기록(CompletedRead)**: 기존 데이터 재사용. 사용자가 완독한 책과 완독 일자. 월별 완독 권수 집계의 원천이며 본 기능이 새로 생성하지 않는다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 목표 설정을 시작해 저장 완료까지 30초 이내에 마칠 수 있다.
- **SC-002**: 홈 진입 후 1초 이내에 이번 달 달성률이 표시된다.
- **SC-003**: 표시되는 완독 권수가 실제 완독 기록과 100% 일치한다(집계 오차 0건).
- **SC-004**: 목표를 설정한 사용자의 60% 이상이 같은 달에 진행 상황을 1회 이상 확인한다.

## Assumptions

- 기존 인증(구글 OAuth + 익명 로그인)과 "다 읽음" 완독 데이터를 재사용한다. 본 기능은 완독 판정 로직을 새로 만들지 않는다.
- 목표 단위는 "권수"이며, 페이지 수·독서 시간 목표는 v1 범위 밖.
- 목표 주기는 "월" 단위만 지원한다(주간·연간 목표는 범위 밖).
- 월 전환 시 목표는 자동 이월하지 않는다(사용자가 매월 새로 설정).
- 익명 세션에서 세운 목표를 정식 계정으로 병합하는 기능은 별도 범위로 둔다.
- 기존 앱과 동일하게 모바일 웹 반응형을 지원한다.
