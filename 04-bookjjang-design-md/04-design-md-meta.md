# 04 · DESIGN.md 생성 메타 프롬프트 (Zero-Loss)

> **이 파일 자체가 "프롬프트"다.** 북박이장(BookJjang) 리포지토리에서 작업 중인 AI 에이전트에게
> 이 문서 전체를 그대로 지시로 주면, 프로젝트 루트에 **`DESIGN.md`** 를 생성한다.
> 목표는 단 하나 — **현재 코드·시안·PRD에 흩어져 있는 디자인 정보를 단 한 조각도 잃지 않고
> (Zero Information Loss) 하나의 정본(正本) 문서로 응축**하는 것.
>
> 사용법: 아래 「지시문」 섹션을 에이전트에게 그대로 전달한다. (이 안내 블록은 참고용이며 전달해도 무방하다.)

---

## 지시문 (여기서부터 에이전트에게 그대로 전달)

당신은 **디자인 시스템 문서화 전문가(Design System Documentarian)** 다.
북박이장(BookJjang) 프로젝트의 **디자인 정본 문서 `DESIGN.md`** 를 작성한다.
이 문서는 **코드가 사라져도 디자인을 그대로 복원할 수 있는 단일 원천(single source of truth)** 이어야 한다.

### 0. 최우선 원칙 — 무손실 (NON-NEGOTIABLE)

1. **값은 원문 그대로.** 모든 색상 HEX, px, ms, 배수, cubic-bezier 값을 **요약·반올림·근사 없이** 그대로 옮긴다. "부드러운 녹색" 같은 서술로 `#4FA557` 을 대체하지 않는다.
2. **전수 열거.** 토큰 하나, prop 하나, variant 하나, size 하나, state 하나, 화면 하나도 빠뜨리지 않는다. "등등", "…", "주요한 것만" 금지.
3. **생략보다 기록.** 코드에 존재하지만 의도·근거가 불명확한 값이 있으면 **삭제하지 말고** 값을 기록한 뒤 `근거: TBD` 로 표시한다.
4. **출처 vs 추론 구분.** 코드/시안에서 직접 확인한 사실과, 당신이 추론·보완한 내용을 명확히 구분해 표기한다(예: 추론 항목엔 _(추론)_ 표시).
5. **한국어 카피 원문 보존.** UI에 등장하는 한국어 문구("아직 서재가 비어 있어요" 등)는 **한 글자도 바꾸지 말고** 그대로 인용한다.
6. **추측 금지.** 소스에 없는 컴포넌트·토큰·화면을 지어내지 않는다. 필요하면 "미정의" 로 남긴다.

### 1. 입력 소스 — 아래를 **모두** 정독한 뒤 착수 (하나도 건너뛰지 말 것)

| 소스 | 무엇을 추출하나 |
|------|----------------|
| `app/globals.css` | **모든** 디자인 토큰: 컬러 램프(green/ink/sage/gray/red/gold) 전부, semantic alias 전부, 타이포 스케일·weight·tracking, spacing 스케일, radius, shadow, motion(ease/duration/press-scale), 폰트 정의, base 스타일 |
| `app/components/ds/*.tsx` + `index.ts` | 각 컴포넌트의 **전체 public API**: 이름, 목적, props(타입·기본값·필수여부), variant, size, state, 내부에서 쓰는 토큰. 대상: `Icon`, `Button`, `IconButton`, `BookBuddy`(마스코트·expression), `TopBar`, `BottomNav`, 그리고 index.ts가 export하는 **모든** 심볼 |
| `app/layout.tsx`, `app/page.tsx`, `app/**` 의 모든 화면 | 화면 조립 방식, 디바이스 프레임(390×844 폰목업, 노치, 상태바, 홈 인디케이터), 레이아웃 구조, 프로필/빈상태 등 각 화면의 **상태(state)별** 구성 |
| `01-prd.md` | 디자인이 충족해야 할 제품 의도·화면 목록·기능(P0/P1/P2)·경험 원칙("나만의 조용한 책장"). 디자인 결정을 이 요구사항과 매핑 |
| `reference1.png`, `rreferece2.png` | **Read 도구로 이미지를 실제로 열어** 하이파이 시안의 레이아웃·색·컴포넌트·마스코트 표현을 관찰하고 코드와 대조. 코드에 아직 없지만 시안에 있는 요소는 "시안에만 존재" 로 명시 |
| `package.json` | 디자인에 영향 주는 의존성(`lucide-react` 아이콘, 차트 라이브러리(Recharts 등), Next/React 버전) |
| `next.config.mjs`, 폰트 파일(`PretendardVariable.woff2`), `public/**` | 폰트 로딩 방식, 정적 에셋 인벤토리 |

> 인접 폴더(`../02-bookjjang_design-system`, `../03-bookjjang-hi-fi`)에 원본 디자인 시스템/시안이 있으면 참고하되, **DESIGN.md는 현재 리포(`04-bookjjang-design-md`)의 실제 구현을 기준**으로 삼는다. 리포와 시안이 다르면 둘 다 기록하고 차이를 명시.

### 2. `DESIGN.md` 필수 목차 — 아래 섹션을 **전부** 포함

각 섹션은 지정된 내용을 **표 중심**으로, 값은 원문 그대로 담는다.

1. **개요 & 디자인 철학** — 제품 한 줄 정의, 브랜드 성격("조용한 개인 책장"), 디자인 원칙(3~6개), 타깃 경험. PRD 6장 "경험 원칙"과 연결.
2. **디자인 원칙(Principles)** — 각 원칙 + 그것이 UI에서 구현되는 방식 1~2문장.
3. **컬러 시스템**
   - 원시 램프 전체 표: 토큰명 · HEX · (색상 스와치 설명). green-50~700, ink-900~300, sage/gray, red, gold **전부**.
   - Semantic alias 전체 표: alias → 참조 램프 → 용도(예: `--brand → green-500`, 버튼/강조).
   - 사용 규칙: "소비자는 semantic alias만 참조, raw 램프 직접 사용 금지" 같은 규칙을 명시.
4. **타이포그래피** — 폰트(Pretendard Variable, weight 45–920 가변), font-family 폴백 체인, weight 토큰(400~800), **타입 스케일 전 항목**(display/h1/h2/h3/body-lg/body/sm/xs — size·line-height·tracking), 사용 예.
5. **레이아웃 & 스페이싱** — spacing 스케일 전부(space-0~16), `--page-gutter`, `--content-max(480px)`, 반응형/브레이크포인트, 폰 프레임 규격(390×844, radius, 노치·상태바·홈인디케이터 수치).
6. **모양(Radius) · 그림자(Shadow/Elevation)** — radius 전부(sm~pill), shadow 전부(xs/sm/md/lg/brand) 원문 값, 언제 어느 elevation을 쓰는지.
7. **모션 & 인터랙션** — ease 토큰(standard/out), duration(fast/base/slow), `--press-scale`, `--focus-ring`, 인터랙션 상태(hover/active/focus/disabled) 규칙, 프레스 피드백.
8. **아이코노그래피** — 아이콘 소스(lucide-react), `Icon` 컴포넌트 API, 사용된 아이콘 이름 목록(search/home/library/settings/user 등 코드에서 발견되는 것 전부), 기본 크기.
9. **컴포넌트 라이브러리** — index.ts가 export하는 **모든** 컴포넌트마다 다음을 서브섹션으로:
   - 목적 / 언제 쓰나
   - Anatomy(구성 요소)
   - **Props 표**: 이름 · 타입 · 기본값 · 필수 · 설명
   - Variants · Sizes · States 전부
   - 사용하는 토큰
   - 코드 위치(`app/components/ds/X.tsx`) 및 대표 사용 예
   - Do / Don't (파악 가능한 범위)
10. **마스코트 — BookBuddy** — 캐릭터 정체성, 지원하는 `expression`(예: sleepy 등 코드의 Expression 타입 값 **전부**), size·blink 등 prop, 사용 맥락(빈 서재에서 낮잠 등), 톤.
11. **화면 명세(Screens)** — 존재하는 각 화면마다: 목적, 레이아웃 구조(위→아래), 사용 컴포넌트, **상태별 구성**(빈 상태/로딩/에러/기본), 마이크로카피 원문. 현재 구현된 화면(예: 빈 서재)과 PRD가 요구하는 화면(로그인/검색/마이페이지/책상세)을 구분해 "구현됨 / 예정" 표기.
12. **패턴(Patterns)** — 내비게이션(BottomNav 3탭 구조·primary 강조), 상단바(wordmark/stats/actions), 별점(1~5 정수, gold), 피드백·빈상태·프로필 표현 등 반복 패턴.
13. **보이스 & 톤 / 마이크로카피** — 한국어 문구의 어조 규칙 + UI에 등장하는 문구 원문 목록(수집한 것 전부).
14. **접근성** — focus-ring, 대비, 터치 타깃, 대체텍스트 등 코드/시안에서 확인되는 항목(없으면 "미확인/권고").
15. **에셋 인벤토리 & 레퍼런스** — 폰트, 이미지(`reference*.png`), public 에셋, 아이콘 소스를 파일 경로와 함께 목록화.
16. **PRD ↔ 디자인 매핑** — PRD의 P0/P1 요구사항 각각이 어떤 화면·컴포넌트로 대응되는지 매핑표. 대응 디자인이 아직 없는 요구사항은 "디자인 필요"로 표시.
17. **미정의 / 열린 질문(Open Questions)** — 근거 TBD로 남긴 값, 코드-시안 불일치, 미구현 화면 등 후속 결정이 필요한 항목 목록.

### 3. 작성 규칙

- **언어**: 서술은 한국어, 식별자·토큰명·코드는 영문 원문 그대로.
- **형식**: 값이 여럿인 것은 반드시 표. 코드 인용은 백틱/코드블록. 파일 참조는 `경로:라인` 형식으로 클릭 가능하게.
- **자기완결성**: DESIGN.md만 읽고도 토큰·컴포넌트·화면을 재현할 수 있어야 한다. "코드 참고" 로 값을 대신하지 말 것.
- **토큰 표기**: semantic alias를 앞세우고 참조하는 raw 램프를 병기(`--brand (= --green-500 = #4FA557)`).
- **위치**: 완성본은 리포 루트 `DESIGN.md` 로 저장.

### 4. 완결성 검증 — **마지막에 반드시 수행**

문서 초안을 쓴 뒤, 다음 자기감사(self-audit)를 실행하고 결과를 **DESIGN.md 맨 끝 「부록: 커버리지 검증」** 에 남긴다:

1. `app/globals.css` 를 다시 스캔해 **모든 CSS 변수**가 문서에 등장하는지 1:1 대조. 누락 토큰 수를 보고(목표 0).
2. `index.ts` 의 export 심볼과 문서의 컴포넌트 섹션을 대조. 누락 컴포넌트 0 확인.
3. 각 컴포넌트 `.tsx` 의 props 인터페이스와 문서 Props 표를 대조. 누락 prop 0 확인.
4. `reference*.png` 관찰 요소 중 문서에 반영 안 된 것 목록화.
5. PRD의 P0 요구사항이 16번 매핑표에 모두 있는지 확인.
6. 검증 결과를 체크리스트(✅/⚠️)로 기록하고, ⚠️ 항목은 17번 열린 질문으로 이관.

검증에서 누락이 발견되면 **보고만 하지 말고 본문을 보완한 뒤** 다시 확인한다.

### 5. 산출물

- 파일: 리포 루트 `DESIGN.md` (Markdown 단일 파일)
- 앞부분에: 문서 목적 1문단 + 최종 목차(TOC)
- 끝부분에: 「부록: 커버리지 검증」 체크리스트
- 톤: 실무 레퍼런스 문서(간결·정확·표 중심). 마케팅 수사 금지.
