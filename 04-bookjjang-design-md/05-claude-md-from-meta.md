# 05 · CLAUDE.md 생성/갱신 메타 프롬프트 — "디자인 작업 전 DESIGN.md 먼저 읽기"

> **이 파일 자체가 "프롬프트"다.** 북박이장(BookJjang) 리포지토리에서 작업 중인 AI 에이전트에게
> 이 문서 전체를 그대로 지시로 주면, 프로젝트 루트에 **`CLAUDE.md`** 를 생성(없으면)하거나
> 갱신(있으면)하여, **모든 디자인 작업 시 먼저 `DESIGN.md`(디자인 정본)를 읽고 진행**하도록
> 강제하는 규칙을 심는다.
>
> 목표는 단 하나 — 에이전트가 색·토큰·컴포넌트·화면을 만지기 전에 **정본 `DESIGN.md`를 반드시
> 참조**하게 만들어, 디자인 일관성과 무손실(Zero-Loss) 원칙이 매 작업마다 지켜지게 하는 것.
>
> 사용법: 아래 「지시문」 섹션을 에이전트에게 그대로 전달한다. (이 안내 블록은 참고용이며 전달해도 무방하다.)

---

## 지시문 (여기서부터 에이전트에게 그대로 전달)

당신은 **리포지토리 온보딩/컨벤션 관리자(Repository Convention Steward)** 다.
북박이장(BookJjang) 프로젝트의 루트 **`CLAUDE.md`** 를 생성하거나 갱신하여,
이 리포에서 일하는 **모든 AI 에이전트가 디자인 관련 작업을 시작하기 전에 반드시
`DESIGN.md`(디자인 정본, single source of truth)를 먼저 읽도록** 만든다.

### 0. 최우선 원칙 (NON-NEGOTIABLE)

1. **무손실(비파괴).** 기존 `CLAUDE.md`가 있으면 **한 줄도 삭제·훼손하지 말 것.** 기존 내용은 보존하고, 새 규칙을 **추가/병합**만 한다.
2. **멱등(Idempotent).** 이미 "DESIGN.md를 먼저 읽으라"는 취지의 규칙이 존재하면 **중복 추가하지 말고**, 문구가 약하거나 모호하면 더 명확하게 **1회만** 갱신한다.
3. **DESIGN.md는 정본.** `DESIGN.md`는 이 리포의 디자인 토큰·컴포넌트·화면·마이크로카피의 **유일한 원천**이다. CLAUDE.md는 이 사실과 "먼저 읽어라"라는 행동 규칙을 명시한다. (DESIGN.md의 내용을 CLAUDE.md에 복제하지 않는다 — 포인터만 둔다.)
4. **명령형·상단 배치.** 규칙은 에이전트가 놓치지 않도록 **CLAUDE.md 상단**의 눈에 띄는 위치에, **강한 명령형**("반드시", "먼저", "하기 전에")으로 적는다.
5. **추측 금지.** 존재하지 않는 파일·경로·스크립트를 지어내지 않는다. 경로는 실제 리포 기준(`DESIGN.md`는 리포 루트).

### 1. 입력 소스 — 착수 전 확인

| 소스 | 무엇을 확인/추출하나 |
|------|----------------------|
| 리포 루트 `DESIGN.md` | **존재 확인**. 이 파일이 디자인 정본임을 전제로 CLAUDE.md가 이를 가리키게 한다. (없으면 아래 §4 예외 처리) |
| 리포 루트 `CLAUDE.md` | **있으면** 전체를 읽고 기존 구조·섹션을 파악해 **병합 위치**를 정한다. **없으면** 새로 생성한다. |
| `app/`, `app/globals.css`, `app/components/ds/*` | "디자인 작업"의 실제 대상 범위를 파악해, 트리거 목록(§2-B)이 리포 구조와 맞는지 확인한다. |
| `01-prd.md`, `04-design-md-meta.md` | 프로젝트 성격·디자인 원칙의 배경 참고(선택). |

### 2. `CLAUDE.md`에 반영할 내용

#### A. 규칙 블록 (반드시 포함 — 아래를 그대로 또는 동등하게 삽입)

CLAUDE.md 상단(제목 바로 아래, 또는 기존 파일이라면 최상단 규칙 영역)에 다음을 넣는다:

```markdown
## 디자인 작업 규칙 (필수)

**이 리포에서 디자인과 관련된 작업을 하기 전에는 반드시 리포 루트의 [`DESIGN.md`](./DESIGN.md)를 먼저 읽는다.**
`DESIGN.md`는 북박이장의 **디자인 정본(single source of truth)** 으로, 모든 컬러/타이포/스페이싱/
radius/shadow/모션 토큰, 컴포넌트 API, 마스코트(BookBuddy), 화면 명세, 마이크로카피가 원문 값으로
정리되어 있다.

- **먼저 읽기:** 아래 "디자인 작업"에 해당하는 요청을 받으면, 코드를 수정하기 전에 `DESIGN.md`의
  관련 섹션을 읽고 그 값·규칙을 따른다.
- **정본 우선:** 색/폰트/여백/모션 값은 임의로 정하지 말고 `DESIGN.md`에 정의된 **semantic alias
  토큰**(`--brand`, `--text-strong`, `--surface-page` 등)을 사용한다. raw 램프 직접 사용 금지.
- **무손실 유지:** 토큰·컴포넌트 값을 변경/추가하면, 같은 변경을 `DESIGN.md`에도 반영해 정본과
  코드가 어긋나지 않게 한다(부록 커버리지 검증 갱신 포함).
- **정본에 없으면:** 필요한 토큰/컴포넌트/화면이 `DESIGN.md`에 없으면 임의 생성하지 말고,
  `DESIGN.md`의 "열린 질문(Open Questions)"을 확인한 뒤 사용자와 결정하거나 정본에 먼저 추가한다.

### "디자인 작업"의 범위 (트리거)
다음 중 하나라도 해당하면 위 규칙을 적용한다:
- `app/globals.css`의 디자인 토큰(컬러/타이포/스페이싱/radius/shadow/모션) 추가·수정
- `app/components/ds/**`의 컴포넌트(생성·props/variant/size/state 변경)
- `app/**`의 화면/레이아웃/폰 프레임/빈 상태 등 UI 구성
- 색상·폰트·아이콘(lucide)·마스코트(BookBuddy) 표현·마이크로카피(한국어 UI 문구) 변경
- 새 화면·컴포넌트·디자인 패턴 추가
```

#### B. 병합 규칙

- **기존 CLAUDE.md가 있을 때:** 위 "디자인 작업 규칙" 섹션을 파일 상단(개요/제목 다음) 또는 기존 "규칙/컨벤션" 영역에 삽입한다. 기존 목차가 있으면 목차에도 한 줄 추가한다.
- **CLAUDE.md가 없을 때:** 새로 만들되, 최소 골격 + 위 규칙을 포함한다. 골격 예:
  - `# CLAUDE.md` 제목 + 1~2문장 프로젝트 소개(북박이장 = 개인 독서 기록 앱; 스택은 Next.js/React — `package.json`·`01-prd.md` 근거)
  - `## 디자인 작업 규칙 (필수)` (위 A 블록)
  - (선택) `## 프로젝트 구조` — `app/`, `app/components/ds/`, `DESIGN.md`, `01-prd.md` 등 핵심 경로 한 줄씩
  - 마케팅 수사 없이 간결하게.

### 3. 작성 규칙

- **언어:** 서술은 한국어, 경로·토큰·식별자는 원문 그대로.
- **경로 표기:** `DESIGN.md`는 클릭 가능한 상대 링크(`[`DESIGN.md`](./DESIGN.md)`)로. 파일 참조는 리포 루트 기준.
- **간결·행동 지향:** CLAUDE.md는 에이전트 지침 문서다. 장황한 설명보다 **행동 규칙**을 앞세운다.
- **정본 복제 금지:** DESIGN.md의 토큰 표 등을 CLAUDE.md로 옮겨 적지 않는다(중복→불일치 위험). 항상 **포인터**만.
- **위치:** 완성본은 리포 루트 `CLAUDE.md`.

### 4. 예외/에지 케이스 처리

- **`DESIGN.md`가 없으면:** 규칙은 그대로 넣되, "`DESIGN.md`가 아직 없다면 먼저 생성/요청하라(`04-design-md-meta.md` 참고)"는 한 줄을 규칙 블록에 덧붙인다. (지금 이 리포에는 `DESIGN.md`가 존재하므로 통상은 해당 없음.)
- **이미 동일 규칙이 있으면:** §0-2에 따라 중복 추가하지 말고, 필요 시 문구만 명확화.
- **다른 CLAUDE 설정 파일:** `.claude/settings.local.json` 등은 건드리지 않는다. 이 작업의 대상은 오직 루트 `CLAUDE.md`.

### 5. 완결성 검증 — 마지막에 반드시 수행

1. 루트에 `CLAUDE.md`가 존재하고, "디자인 작업 전 `DESIGN.md`를 먼저 읽는다"는 **명령형 규칙**이 상단에 있는지 확인.
2. `DESIGN.md`로 향하는 **상대 링크**가 정확한지(클릭 가능) 확인.
3. "디자인 작업"의 **트리거 목록**(globals.css 토큰 / ds 컴포넌트 / 화면 / 색·폰트·마스코트·카피)이 포함됐는지 확인.
4. 기존 CLAUDE.md 내용이 있었다면 **한 줄도 손실되지 않았는지** 대조.
5. 규칙이 **중복 없이 1회**만 존재하는지 확인.
6. 검증 결과를 사용자에게 체크리스트(✅/⚠️)로 간단히 보고.

누락·손실이 발견되면 **보고만 하지 말고 보완한 뒤** 다시 확인한다.

### 6. 산출물

- 파일: 리포 루트 `CLAUDE.md` (생성 또는 무손실 갱신)
- 포함: 상단의 "디자인 작업 규칙 (필수)" 섹션 + 트리거 목록 + `DESIGN.md` 상대 링크
- 톤: 간결한 에이전트 지침 문서. 마케팅 수사 금지.
- 마지막에: 무엇을 추가/변경했는지 1~3줄 요약 보고.
