## 1. 요약 (Executive Summary)

**북박이장**은 읽은 책을 한곳에서 체계적으로 기록하고, 독서 과정에서 느낀 생각과 메모를 함께 저장하며, 독서량과 습관을 시각적으로 확인할 수 있는 **개인 독서 기록 서비스**다. "book(책) + 붙박이장"이라는 이름처럼, 사용자가 자신만의 온라인 책장에 책을 차곡차곡 쌓아가는 경험을 제공하여 꾸준한 독서 습관 형성과 나만의 독서 아카이브 구축을 돕는다.

---

## 2. 배경 및 맥락 (Background & Context)

**문제 상황**
- 책을 읽고 나면 내용과 감상이 흩어져 기록이 남지 않고, 시간이 지나면 무엇을 읽었는지조차 잊게 된다.
- 독서 기록이 메모 앱, 종이 노트, SNS 등 여러 곳에 분산되어 있어 한눈에 모아 보기 어렵다.
- 자신이 얼마나 읽었는지, 어떤 독서 패턴을 가졌는지 확인할 방법이 없어 동기 부여가 약하다.

**사용자**
- 읽은 책을 기록하고 독서 습관을 만들고 싶은 사용자
- 독서 기록을 한곳에서 체계적으로 관리하고 싶은 사용자

**시장/유사 서비스**
- 굿리즈(Goodreads), 북적북적, 리더스, 밀리의 서재 기록 기능 등 유사 서비스가 존재하나, 대부분 소셜/커머스 중심이거나 무겁다. 북박이장은 **"나만의 조용한 책장"**이라는 개인 아카이브 경험에 집중한다.

**왜 지금인가**
- 개인 독서 관리 도구에 대한 니즈가 꾸준하며, 구글 로그인 + 공개 도서 검색 API로 빠르게 MVP를 만들 수 있는 환경이 갖춰져 있다.

## 3. 목표 및 성공 지표 (Objectives & Success Metrics)

### 목표 (Goals)
1. 사용자가 검색 → 서재 추가 → 독서 기록까지 **3분 이내**에 첫 책을 기록할 수 있게 한다.
2. 읽은 책과 감상(메모·한줄평·별점)을 한곳에 모아 **손실 없이 영구 보관**한다.
3. 독서 통계를 시각화하여 사용자가 자신의 **독서량과 습관을 즉시 확인**할 수 있게 한다.
4. 재방문을 유도하여 **꾸준한 독서 아카이브**가 쌓이는 경험을 제공한다.

### 비목표 (Non-Goals) — 이번 범위에서 제외
1. **소셜/팔로우 기능** — 타 사용자 서재 구독, 댓글, 좋아요. (개인 아카이브에 집중, 추후 검토)
2. **전자책 뷰어/본문 열람** — 우리는 "기록" 도구이지 "읽기" 도구가 아니다.
3. **도서 구매/커머스 연동** — 결제, 장바구니 등.
4. **AI 추천/독서 챌린지** — MVP 이후 성장 기능으로 보류.
5. **모바일 네이티브 앱** — 우선 반응형 웹으로 검증.

### 성공 지표 (Success Metrics)
| 지표 | 현재 | 목표 | 측정 방법 |
|------|------|------|-----------|
| 신규 가입 후 첫 책 기록 완료율(activation) | — | 60% | 가입 후 24시간 내 서재에 책 1권 이상 추가 |
| 주간 재방문율(WAU/가입자) | — | 30% | 로그인 이벤트 기준 주간 집계 |
| 사용자당 평균 기록 책 수 | — | 5권 / 첫 달 | 서재 등록 도서 수 |
| 독서 기록 작성률 | — | 70% | 서재 추가 도서 중 메모/한줄평/별점 1개 이상 작성 비율 |

## 4. 대상 사용자 및 세그먼트 (Target Users & Segments)

**핵심 세그먼트**
- **기록형 독자**: 책을 읽고 감상과 생각을 남기고 싶지만 마땅한 도구가 없던 사용자. 메모 앱이나 종이 노트를 임시로 사용 중.
- **습관 형성 독자**: 독서량을 늘리고 싶고, 자신의 진척을 시각적으로 보며 동기를 얻고 싶은 사용자.

**대표 사용자 여정**
> "이번 달에 읽은 책이 몇 권이더라?" → 북박이장 접속 → 통계 그래프 확인 → 읽는 중인 책의 진행률 업데이트 → 오늘 읽은 부분의 메모 추가

## 6. 솔루션 개요 (Solution Overview)

**페이지 구조**
1. **로그인 페이지** — 구글 로그인 단일 진입점.
2. **책 검색 페이지** — 검색창 → 결과 리스트 → 상세 → "내 서재에 추가".
3. **마이페이지(내 서재)** — 프로필(별명/이미지) · 독서 통계 · 읽은 책 목록.
4. **독서 기록(책 상세)** — 메모 · 한줄평 · 별점 · 시작일/완료일 · 진행률.

**주요 설계 결정**
- **도서 데이터**: **알라딘 도서 검색 API(Aladin Open API)** 연동으로 확정. 상품 검색(ItemSearch)·상품 조회(ItemLookUp) 활용, 표지/제목/저자/출판사/출간일/ISBN/소개를 응답에서 수집. TTB(Aladin) API 키 발급 필요.
- **인증**: 구글 OAuth 2.0 단일 제공. 이메일/비밀번호 미지원(가입 마찰 최소화).
- **데이터 모델(초안)**: `User`(id, 별명, 프로필이미지) — `LibraryBook`(user_id, book_meta, 상태, 별점(1~5), 한줄평, 메모, 시작일, 완료일, 진행률). 메모는 책당 1개이므로 `LibraryBook`의 단일 필드로 관리(별도 테이블 불필요). 도서 메타는 알라딘 API 응답을 캐싱 저장.
- **경험 원칙**: 소셜 압박 없는 "조용한 개인 책장". 빠른 기록, 부담 없는 UI, 표지 중심의 시각적 책장.

**기술 스택** *(확정)*
- **플랫폼**: 반응형 웹 우선 (모바일/데스크톱 대응). 네이티브 앱은 향후 검토.
- **프런트엔드**: Next.js (React) — 표지 중심 책장 UI, SSR/이미지 최적화 활용.
- **백엔드/인증/DB/스토리지**: **Supabase** — 구글 OAuth 인증, PostgreSQL(관계형 DB), 프로필 이미지 스토리지를 한 번에 제공하여 MVP 구축 속도 확보.
- **도서 API**: 알라딘 Open API는 **서버 사이드(API 라우트)에서만 호출**하여 TTB 키를 노출하지 않음.
- **호스팅**: Vercel(프런트) + Supabase(백엔드) 관리형 조합.
- **차트**: 통계 시각화는 경량 차트 라이브러리(예: Recharts) 사용.
> 참고: 위 스택은 1인/소규모 개발과 빠른 MVP에 최적화한 선택이며, 팀 상황에 따라 대체 가능(예: 백엔드를 Node/Express + 직접 OAuth로 분리).

## 8. 일정 및 단계 (Timeline & Phasing)

| 단계 | 범위 | 산출물 |
|------|------|--------|
| **Phase 1 — MVP** | 구글 로그인, 책 검색·상세, 서재 추가, 별점/한줄평/메모, 읽은 책 목록, 기본 통계 (P0) | 핵심 기록 흐름이 동작하는 서비스 |
| **Phase 2 — 기록 심화** | 시작일/완료일, 진행률, 독서 상태 분류, 별명/프로필 변경, 서재 정렬·필터 (P1) | 상세한 독서 관리 경험 |
| **Phase 3 — 동기부여** | 독서 목표, 장르 통계, 내보내기, 스트릭 (P2) | 습관 형성·재방문 강화 기능 |

**의존성**
- Phase 1은 외부 도서 API 확정과 구글 OAuth 설정에 선행 의존.
- 통계(P0)는 최소 지표로 시작하고, 시작일/완료일(P1) 확보 후 기간별 통계를 고도화.
